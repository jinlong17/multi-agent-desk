点击组件查看“它拥有什么、它绝不能做什么”。核心不变量是 Device Daemon 持有秘密与 Provider 进程，Control Plane 只存元数据和密文。

安全、配对与凭据授权

Passkey 解决“你是谁”，Device Key 解决“谁能解密”。Control Plane 即使被读取，也不能替换已 pin 的公钥或解密 CredentialBundle。

1. Bootstrap Ceremony创建用户、注册 Passkey，并登记一台具有 OS Vault 的 Daemon/Desktop 为初始 Trust Anchor。
2. Device Keys每个 Daemon/Web/Desktop 生成 Ed25519 + X25519；Web 安全能力不足时降级 metadata-only。
3. 指纹 + Attestation双端核对 120-bit Base32 指纹；已 pin 设备签署 DeviceAttestation，Server 不能伪造。
4. Credential Grant来源 Daemon 只向已 pin/有效 attestation 的目标 Daemon发送 E2EE CredentialBundle。
5. Session Key每个 Session 独立密钥，分别包装给已批准客户端；AEAD AAD 绑定路由元数据。
6. 撤销与轮换撤销设备后关闭 WSS、轮换所有受影响活动 Key，并只重新分发给存活设备。
必须诚实披露的残余风险

目标服务器获得授权后，在凭据有效期内必然可以使用或复制它。设备撤销不能远程擦除攻击者已经复制的秘密；需要同时在 Provider 官方安全页面撤销会话或重置凭据。
