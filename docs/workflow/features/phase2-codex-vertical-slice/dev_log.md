## Risks and Blockers

- P2A verified the materialization manager contract, not a production
  credential source. P2B now freezes the smallest executable portable Vault v1
  and exact interactive enrollment transaction; OS keyring wrapping and grants
  remain a later migration rather than an implied dependency.
- Vault first use is no longer implicit: `0005` creates empty schema and exactly
  one authenticated initializer can bind it to a password. Corrupt singleton
  state fails closed; rekey/password change remains explicitly outside Phase 2.
- P3A now owns the previously missing daemon registration and one shared
  CredentialRuntime per CredentialInstance, with per-Session bindings,
  one-reader JSON-RPC, daemon-derived mapping, exact Approval dispatch, and
  idempotent finalization. No Linux verification begins before P3A verifies.
- Codex conversation resize is frozen as typed unsupported. Only a future exact
  command-execution process contract may enable `command/exec/resize`.
- P0 is now a mandatory implementation phase: the current Fake-only schema,
  Store validators, local IPC authorizer, and CLI cannot be bypassed by a
  provider adapter.
- The Provider Spike's short two-device run is not a multi-writer safety claim;
  ADR 0014's single-writer/CAS boundary is mandatory.
- A credentialed Linux environment and a second local CLI are required for the
  live exit. Without them, feature-verify must preserve a reproducible BLOCKED
  result rather than downgrade the acceptance criterion.
- Windows remains covered by the Phase 1 CLI/Daemon/IPC and CI baseline, but
  this feature does not establish real Windows Codex compatibility.
- Approval payload redaction fields and the Linux schema/version remain evidence
  gates; unmapped or unverified fields stay unavailable.
- Approval cancel is a Provider mutation with a durable
  `cancel/cancelled/written` result; it follows the same claim/write/ambiguity
  and idempotency rules as approve/deny rather than being a local-only label.
- Resume semantics are frozen: a new local Session is created only after exact
  continuation evidence; otherwise `provider_resume_unsupported` returns with
  zero Provider frame, zero new Session, zero source transition, and no recovery
  claim. The operator may separately start an unrelated Session.
